ArcGIS Runtime und MVVM

MVVM

Das Model-View-ViewModel kurz MVVM-Pattern ist ein Vertreter einer ganzen Familie von Pattern deren vorrangige Aufgabe es ist, den Content von Daten von deren Darstellung zu trennen. Die Ursprünge dieses Pattern liegen schon 30 Jahre zurück im Smalltalk als Model-View-Controller (MVC) und fand über die Weiterentwicklung Model-View-Presenter (MVP) und Supervisor zur aktuellen Ausprägung. Dabei bezeichnet MVVM eine spezielle Ausprägung des Presentation-Pattern durch Microsoft in deren neueren Technologien WPF und Silverlight. Wie man sieht, ist MVVM universell gegenüber der Plattform und das macht auch den eigentlichen Charme aus.
Die Idee dahinter ist die Kappselung der Geschäftlogik in einem eigenen Konzeot (Model) deren Daten mittels eines Vermittler (ViewModel) im Frontend (View) dargestellt werden. Diese Art der Aufteilung adressiert gleich mehrere Aufgaben
  • Fokussierung der technichen Verantwortlichkeiten
    Die Geschäftslogik kümmert sich ausschließlich um das Management von Daten. Das Frontend hingegen ist lediglich zur Anzeige und Manipulation von Daten verantwortlich. Damit ist auch den Architekturpattern SRP und SOC Rechnung getragen.
  • Fokussierung der organisatorischen Verantwortlichkeiten
    Währenddessen der Entwickler die eigentliche Programmlogik programmiert, kann das Frontend völlig losgelöst von einem UX-Experten designet werden.
  • Flexibilität
    Eine derartige Architektur erlaubt das schnelle Auswechseln und Anpassen von Komponenten. Was heute noch auf dem Desktop ist, kann morgen schon einfach ins Web portiert werden. Dazu müssen entsprechend nur der View mit seinem zugehörigen ViewModel implementiert werden. Die meist aufwändige Businesslogik bleibt unangetastet.
  • Testbarkeit
    Durch einen fehlenden UI-Bezug ist die Geschäftslogik relativ einfach Unit-testbar
Das Besondere an MVVM ist das sogenannte Databinding. Dabei werden die UI-Elemente vom Designer einfach auf die entsprechenden Anknüpfpunkte des ViewModels geklemmt. Die Technologie sorgt dann völlig eigenständig dafür, dass die Daten korrekt im Frontend angezeigt werden. So kann beispielsweise ein Integerwert mit einem Slider, Spinedit oder Eingabefeld verbunden werden. Als Entwickler muss ich mir keine Gedanken mehr machen den Slider oder das Spinedit korrekt zu setzten - das passiert automtisch im Hintergrund durch das Binding für mich.

Werkzeuge

Expression Blend

Microsoft bietet mit Expression Blend ein ausgezeichnetes Programm zum Designen von Oberflächen an. Die anvisierte Zielgruppe wird schon auf dem ersten Blick klar: Designer. Visuall orientiert sich Expression Blend an den Adobe-Klassikern - ein dunkles UI um die Augen nicht abzulenken und unzählige kleine Controls zur Parametrierung. Aber nicht nur dem Designer wird die Arbeit erleichtert, auch dem Entwickler kann ExpressionBlend eine gute Hilfe sein.

MVVM Light Toolkit

Darüber hinaus gibt es für die Entwicklung mit MVVM unzählige kleine und große Helferlein. Ein von uns sehr gern verwendetes Framework ist das kostenfreie MVVM Light Toolkit von Galasoft.
Die für uns interessantesten Punkte sind
  • umfangreichere Bindingmöglichkeiten (Blendability) mittels EventToCommand
  • Verwendbarkeit für WPF- und Silverlight-Projekte
  • Messaging-Unterstützung
  • Umfangreiche Dokumentation
Mittels NuGet lässt sich das Toolkit einfach in eigene Projekte einbinden. Dabei wird neben den Einbetten der eigentlichen Bibliotheken auch der DevTree für die Verwendung angepasst.
